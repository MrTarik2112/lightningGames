# Plan: Pencere Boyutunu %25 Büyütme

## Hedef
Uygulama penceresini ve oyun canvas boyutlarını %25 oranında büyütmek. Oyunları bozmadan ölçeklemek.

## Mevcut Boyutlar
- **Pencere**: 960x700
- **Canvas**: 880x540

## Yeni Boyutlar (%25 artış)
- **Pencere**: 1200x875 (960×1.25=1200, 700×1.25=875)
- **Canvas**: 1100x675 (880×1.25=1100, 540×1.25=675)

## Değişiklik Yapılacak Dosyalar

### 1. main.js (Satır 130-131)
```javascript
// Önce:
width: 960,
height: 700,

// Sonra:
width: 1200,
height: 875,
```

### 2. renderer/gameManager.js (Satır 376-377)
```javascript
// Önce:
const baseWidth = game.meta.canvasWidth || 880;
const baseHeight = game.meta.canvasHeight || 540;

// Sonra:
const baseWidth = game.meta.canvasWidth || 1100;
const baseHeight = game.meta.canvasHeight || 675;
```

### 3. renderer/launcher.js (Satır 2820-2830)
Tüm oyun kayıtlarında canvasWidth ve canvasHeight değerlerini güncelle:
- 880 → 1100
- 540 → 675

### 4. styles/main.css
Gerekirse flex container'a max-width eklemek için kontrol et (muhtemelen gerekmez).

## Ölçekleme Mantığı
- Oyunlar `canvas.width` ve `canvas.height` değerlerini kullanarak konumlandırma yapıyor
- Çoğu oyun göreceli (relative) konumlandırma kullanıyor
- Canvas boyutu arttığında oyunlar otomatik olarak yeni boyuta göre ölçeklenecek
- `renderScale` ayarı kullanıcıların kalite/performans dengesini ayarlamasına izin veriyor

## Risk Analizi
- Düşük: Çoğu oyun canvas boyutuna göre dinamik çalışıyor
- Orta: Başlangıç konumları veya hız değerleri canvas boyutuna bağlıysa etkilenebilir
- Önlem: Test aşamasında birkaç oyun test edilecek

## Sonraki Adımlar
1. main.js boyutlarını güncelle
2. gameManager.js base boyutlarını güncelle  
3. launcher.js'deki tüm oyun kayıtlarını güncelle
4. Test et (Ctrl+Alt+G ile launcher açıp oyunları test et)
